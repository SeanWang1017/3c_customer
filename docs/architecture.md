# 3C 电商智能客服 Agent 系统 — 架构说明

> 本文档为对外技术说明，包含系统架构图、核心算法和模块说明。

---

## 1. 系统概述

本系统是一个面向 3C 电商场景的智能客服 Agent，采用**本地小模型分类意图 + 云端大模型生成回复 + 客服人工审核后发出**的双 LLM 架构。

### 1.1 核心特性

| 特性 | 说明 |
|:----|:-----|
| 意图分类 | Qwen2.5-0.5B + QLoRA 本地微调，5 分类，89.8% 准确率 |
| 回复生成 | qwen-flash 云端 API，配合 RAG 检索和工具调用结果 |
| 多轮对话 | LangGraph + InMemorySaver，支持会话隔离和代词理解 |
| 兜底机制 | 4 重保险架构，任意一层失效系统仍可用 |
| 人工审核 | 客服可确认/修改/驳回 AI 生成的回复建议 |

## 3. 核心算法

### 3.1 4 重保险意图识别架构

本系统的核心创新是**4 重保险**意图识别架构，确保任意一层失效系统仍能保持端到端可用。

```text
用户输入
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 第 1 重保险：本地 LoRA 模型                                       │
│   - 模型：Qwen2.5-0.5B + QLoRA (int4)                          │
│   - 训练：2028 条意图分类数据，3 epoch                           │
│   - 输出：5 分类之一 + 置信度                                     │
│   - 准确率：89.8% (220/245)                                     │
└─────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 第 2 重保险：关键词规则后处理                                      │
│   - 触发条件：intent ∈ {product_intro, policy_qa} 或低置信度    │
│   - 规则：17 条正则表达式                                        │
│   - 作用：消解 product_intro ⇄ policy_qa 互混                   │
│   - 效果：3/5 端到端 query 触发，全部改判正确                    │
└─────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 第 3 重保险：云端 LLM 复核（可选）                                 │
│   - 触发条件：confidence < 0.7 且未被第 2 重改判                 │
│   - 模型：qwen-flash (云端 API)                                  │
│   - 作用：本地模型不确定时，调云端重新分类                        │
│   - 成本：一次云端调用 (~500ms)                                  │
└─────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 第 4 重保险：人工审核                                             │
│   - 位置：FastAPI 客服工作台                                      │
│   - 操作：确认 / 修改 / 驳回                                      │
│   - 作用：最终质量保证                                            │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 意图分类算法

#### 3.2.1 模型微调

- **基座模型**：Qwen2.5-0.5B (540M 参数，942MB)
- **微调方法**：QLoRA (int4 量化 + LoRA)
- **训练参数**：
  - LoRA rank=16, alpha=32, dropout=0.05
  - batch_size=2, gradient_accumulation=4 (有效 batch=8)
  - learning_rate=3e-5, epoch=3, cutoff_len=512
  - warmup_steps=50, cosine scheduler
- **训练数据**：2028 条意图分类样本 (alpaca 格式)

#### 3.2.2 意图类别

| 意图 | 说明 | 示例输入 |
|:-----|:-----|:---------|
| order_query | 订单/物流查询 | "帮我查一下订单 JD20240610001" |
| product_intro | 商品咨询/介绍 | "这款耳机降噪怎么样？" |
| product_recommend | 商品推荐 | "推荐一款 3000 元以内的游戏手机" |
| policy_qa | 售后政策问答 | "耳机拆封后还能退吗？" |
| ticket_transfer | 转人工/投诉 | "我要投诉，转人工！" |

#### 3.2.3 推理流程

```python
# 1. 加载 LoRA 模型
model = load_model_with_lora("saves/qwen2.5-0.5b-intent-lora/checkpoint-400")

# 2. 构造 prompt（与训练格式一致）
system_prompt = "You are Qwen, a customer service intent classifier."
user_prompt = f"对用户问题进行意图分类\n{user_query}"

# 3. 模型推理
output = model.generate(system=system_prompt, user=user_prompt)

# 4. 提取意图标签
intent = extract_intent(output)  # 从生成文本中提取 5 分类之一
confidence = calculate_confidence(output)  # 基于 token 概率计算置信度
```

### 3.3 RAG 检索算法

#### 3.3.1 向量库构建

- **Embedding 模型**：BGE-small-zh-v1.5 (中文语义向量化)
- **向量库**：FAISS IndexFlatIP (内积相似度)
- **数据源**：
  - FAQ 库：33 条售后政策 (0.1MB)
  - 商品库：3929 条 3C 商品 (7.7MB)

#### 3.3.2 检索流程

```text
用户问题
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ BGE 编码                                                        │
│   - 输入：用户问题文本                                           │
│   - 输出：768 维语义向量                                         │
└─────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ FAISS 检索                                                      │
│   - FAQ 库：top-3 相似文档                                      │
│   - 商品库：top-3 相似商品                                      │
│   - 相似度：内积 (Inner Product)                                 │
└─────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 结果拼接                                                        │
│   - 格式：[FAQ 结果] + [商品结果]                               │
│   - 作为 LLM 上下文输入                                         │
└─────────────────────────────────────────────────────────────────┘
```

### 3.4 工具调用算法

#### 3.4.1 工具列表

| 工具名 | 功能 | 输入 | 输出 |
|:-------|:-----|:-----|:-----|
| query_order | 订单查询 | order_id (JD+11位数字) | 订单状态 JSON |
| search_faq | FAQ 检索 | 查询文本 | 相似 FAQ 列表 |
| search_product | 商品检索 | 查询文本 | 相似商品列表 |
| create_ticket | 工单创建 | 对话历史 | 工单号 + 通知状态 |

#### 3.4.2 工具调用流程

```text
意图分类结果
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 意图 → 工具映射                                                 │
│   - order_query → query_order                                  │
│   - product_intro → search_product                             │
│   - product_recommend → search_product                         │
│   - policy_qa → search_faq                                     │
│   - ticket_transfer → create_ticket (跳过 LLM)                 │
└─────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ LLM 决策                                                       │
│   - qwen-flash 根据意图选择工具                                 │
│   - 提取工具参数（如 order_id, query 文本）                      │
│   - 调用工具获取结果                                            │
└─────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 回复生成                                                        │
│   - 输入：用户问题 + 工具返回数据                                │
│   - 输出：自然语言回复建议                                      │
└─────────────────────────────────────────────────────────────────┘
```

### 3.5 多轮对话算法

#### 3.5.1 会话管理

- **状态管理**：LangGraph + InMemorySaver
- **会话隔离**：thread_id 按用户隔离对话历史
- **记忆机制**：保留最近 N 轮对话历史

#### 3.5.2 代词理解

```text
用户："帮我查一下订单 JD20240610001"
AI：订单状态为已发货...
用户："它到哪了？"  ← "它" = 订单 JD20240610001
```

- **实现**：LLM 根据对话历史理解代词指代
- **验证**：端到端测试 5/5 通过

### 4.2 关键文件说明

| 文件 | 功能 | 核心算法 |
|:-----|:-----|:---------|
| `intent/classifier.py` | 本地意图分类 | QLoRA 推理 + 置信度计算 |

## 5. 数据流

### 5.1 端到端数据流

```text

    │
    ▼
[1] 意图识别

    │
    ▼
```
