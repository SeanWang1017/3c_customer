# 3C 电商客服对话数据生成指南

## 任务

根据商品数据 `jd_all_cleaned.xlsx`，生成 3000-5000 条 3C 电商客服对话数据。

## 数据格式

```json
{
  "messages": [
    {"role": "user", "content": "用户问题"},
    {"role": "assistant", "content": "客服回答"},
    {"role": "user", "content": "用户追问"},
    {"role": "assistant", "content": "客服回复"}
  ],
  "scenario": "场景标签",
  "product": "商品名称",
  "category": "商品品类"
}
```

- 每条 **2-4 轮**对话，用户和客服交替发言
- 用户语言 **口语化**，可带错别字、语气词
- 场景标签：`product_intro` / `product_recommend` / `order_query` / `policy_qa` / `ticket_transfer`

## 商品数据字段

| 字段 | 示例 |
|:----|:------|
| title | 雷蛇战锤狂鲨V3有线入耳式耳塞 |
| price | 339.0（元） |
| sales | 20000.0 |
| rating | 0.97 |
| store | 雷蛇（Razer）京东自营旗舰店 |
| category | Earphone |

**品类分布：** Phone(728) | Computer(676) | Earphone(515) | Smart_Watch(509) | Camera(504) | Mouse_Keyboard(488) | Speaker(257) | Smart_Glasses(252)

## 重要规则：哪些必须真实，哪些可以生成

| 内容 | 规则 | 说明 |
|:----|:----|:------|
| **商品名称** | ✅ 必须从数据源中选取，不能编造 | 否则 RAG 检索到的商品和训练数据对不上 |
| **价格/销量/评分** | ✅ 必须使用数据源中的真实值 | 客服不能乱报价 |
| **客服回答** | ⚠️ 可以模板化、通用表达 | 重点学对话流程，不是背数据。最终系统会靠 RAG 检索真实信息来回答 |
| **用户问题** | ⚠️ 模板+口语化包装 | 同一个商品覆盖多种问法 |

## 5 个场景

### 1. product_intro（产品介绍，20%）

用户咨询商品参数、价格、功能等。客服基于商品实际数据回答。

```
用户：这个电脑是多大内存的？
客服：这款AMD迷你主机搭载的是32GB DDR5内存，1TB固态硬盘。
用户：能自己加内存吗？
客服：支持扩展的，有两个内存插槽，最高支持64GB。
```

### 2. product_recommend（产品推荐，20%）

用户根据需求寻求推荐。客服基于价格、销量、评分推荐。

```
用户：我想买个玩游戏用的耳机，预算500以内
客服：推荐这款雷蛇战锤狂鲨V3，339元，低延迟，适合游戏，销量2万+，好评97%。
用户：这个带麦克风吗？
客服：带麦克风，通话清晰，开黑没问题。
```

### 3. order_query（订单查询，20%）

用户查询订单状态、物流、配送时间。客服索要订单号后回复。

```
用户：我的手机什么时候到？下单三天了
客服：请问您的订单号是多少？
用户：JD20240615001
客服：订单已到达当地配送站，预计今天下午送达。
```

### 4. policy_qa（政策问答，20%）

用户咨询退换货、保修、发票等政策。客服参照三包政策回答。

```
用户：这个耳机保修多久？
客服：7天无理由退货，15天质量问题换货，整机保修1年。
用户：用了两个月不响了能修吗？
客服：保修期内免费维修，上门取件，来回运费我们承担。
```

### 5. ticket_transfer（工单转人工，10%）

用户投诉或要求转人工。客服安抚情绪后创建工单。

```
用户：换了两次还有毛病，给我转人工！
客服：马上为您创建工单转接专员。请问订单号？
用户：JD20240315008，我要投诉！
客服：已创建工单WT20240615001，升级至主管处理，24小时内联系您。
```

## 输出要求

- 总条数：3000-5000 条
- 格式：JSON 文件
- 场景比例：product_intro(20%) / product_recommend(20%) / order_query(20%) / policy_qa(20%) / ticket_transfer(10%)
- **商品名称、价格、评分等必须使用数据源中的真实值，不能编造**
- 每条对话需标记对应的场景、商品名、品类
